## Project Overview

This project investigates whether structural properties of a heterogeneous knowledge graph can predict relationships between anime titles. Using metadata from the AniList GraphQL API, we build a knowledge graph where anime, genres, studios, and source materials are distinct node types. We then apply three classical link-prediction heuristics and a logistic regression relation classifier to score the probability that two anime share a typed relationship such as SEQUEL, PREQUEL, or SPIN_OFF.

### Research Questions

1. Can attribute-based heuristics predict SEQUEL/PREQUEL relations between anime?
2. Can a classifier distinguish the type of relation (continuation vs. side vs. alternative)?
3. How does Preferential Attachment compare to overlap-based methods?

## How to Run

The pipeline has four steps. Run them in order. Steps 1 and 2 only need to be run once since their outputs are cached to disk.

### Step 1 — Data Collection

Fetches 500 anime from the AniList GraphQL API and saves them to `data/anime_data.json`. Skipped automatically on subsequent runs if the cache file already exists.

```bash
python data_collection.py
```

Expected output:
```
[data_collection] Fetching up to 500 anime from AniList API...
[data_collection] Page 1/10 – collected 50 anime so far.
...
[data_collection] Saved 500 anime to data/anime_data.json
```

> The `data/anime_data.json` file is already included in the repository. You only need to run this step if you want to re-fetch fresh data, or if the file is missing.

---

### Step 2 — Build the Graph

Reads `data/anime_data.json`, constructs the heterogeneous knowledge graph and the anime-only projection graph, prints statistics, and saves the result to `data/graph_data.json`.

```bash
python graph_builder.py
```

Expected output:
```
[graph_builder] Loaded 500 anime.
[graph_builder] Heterogeneous KG — 626 nodes, 3146 edges
[graph_builder] Typed relation edge counts: {'SEQUEL': 151, 'PREQUEL': 151, ...}
[graph_builder] Projection graph (min_weight=4) — 500 nodes, 3803 edges
[graph_builder] Evaluation triples available: 327
[graph_builder] Graph data saved to data/graph_data.json
```

> `data/graph_data.json` is already included. Only re-run this step if you change the graph construction logic or want a different `min_weight` threshold.

---

### Step 3 — Run Evaluation

Loads `data/graph_data.json` and runs the full link prediction evaluation pipeline: stratified train/test split, negative sampling, scoring with all three heuristics, and training the relation classifier. Saves results to `results/eval_results.json`.

```bash
python evaluate.py
```

Optional arguments:
```bash
python evaluate.py --graph data/graph_data.json   # custom graph path
python evaluate.py --test-ratio 0.2               # fraction held out for testing (default 0.2)
python evaluate.py --seed 42                      # random seed (default 42)
python evaluate.py --neg-mult 5                   # negatives per positive (default 5)
```

Expected output:

## Key Design Decisions

**Why min_weight = 4 in the projection graph?**
At min_weight = 2, density was 0.54 — almost every anime connected to every other one, making link prediction trivial and meaningless. Raising the threshold to 4 (requiring at least 4 shared genres or studios) reduces density to 0.03, which is appropriate for link prediction.

**Why is source medium excluded from projection weights?**
Approximately 42% of anime in the dataset share the "MANGA" source. Including it would add +1 to nearly half of all pairs regardless of actual similarity, inflating density artificially. Source is still a node type in the full heterogeneous KG and is used as a feature in the relation classifier.

**Why are relation edges excluded from the projection graph?**
SEQUEL, PREQUEL, and other typed Anime-Anime edges are the evaluation targets. Including them in the projection graph would contaminate the training signal and make evaluation invalid. They are held out entirely as ground truth positives.
